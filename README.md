## Задание по курсу "Введение в профессиональную деятельность" Головатенко Алёна АСУбз-23-1

### Тема: Система управления меню ресторана
---
### 1. Постановка Задачи

* **Проблема:** Отсутствие структурированной и гибкой системы для работы с ассортиментом блюд.
* **Цель:** Практическое применение **ООП** (наследование, полиморфизм, инкапсуляция) для создания консольного приложения, управляющего объектами меню.

---

### 2. Инструменты и реализация

* **Инструмент проектирования:** UML-диаграмма классов.
* **Тип ПО:** Консольное приложение.
* **Контроль версий:** Реализация выполнена с использованием **Git** и размещена на **GitHub**.
* **Язык программирования:** Java.
* **Используемое ПО** IntelliJ IDEA.
#### Ключевые Принципы ООП

1.  **Наследование и Абстракция:**
    * Создан **абстрактный класс** `Dish`.
    * Реализована иерархия: `Dish` → `Appetizer`, `MainCourse`, `Dessert`.
2.  **Полиморфизм:** Методы, такие как `isAllergenFree()`, **переопределены** в классах-наследниках для реализации специфической логики проверки на аллергены (глютен, орехи, лактоза).
3.  **Инкапсуляция:** Доступ к данным строго контролируется через методы `get*`/`set*`.
4.  **Агрегация:** Класс `RestaurantMenu` управляет всей коллекцией блюд.
5.  **Дополнительно (Сериализация):** Реализованы сохранение и загрузка меню (`FileWork`).

---

### 3. Функционал Приложения

Проект — интерактивное консольное меню, позволяющее:

| Действие | Описание |
| :--- | :--- | 
| **Добавить/Удалить** | Управление объектами блюд в коллекции. |
| **Посмотреть меню** | Вывод полной информации о всех блюдах. |
| **Поиск** | Найти блюдо по названию. |
| **Блюда для аллергиков** | Фильтрация блюд|
| **Сохранение/Загрузка** | Сохранение и восстановление состояния меню из файла. |

---

### 4. Результат

Проект **успешно реализован**. Поставленная задача решена: создана гибкая, расширяемая система, демонстрирующая практическое применение всех основных принципов **ООП** (наследование, полиморфизм, инкапсуляция, агрегация) и использование **GitHub** для контроля версий.
