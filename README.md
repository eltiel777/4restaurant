## Система Управления Меню Ресторана

### 1. Постановка Задачи

**Проблема:**: создание структурированной и расширяемой системы для управления ассортиментом блюд в ресторане.
**Цель:** практическое применение ключевых принципов ООП для реализации системы, управляющей ассортиментом блюд.
---
#### 2. Реализованные Принципы ООП

1.  **Наследование и Абстракция:** * Создан **абстрактный класс** `Dish` (Блюдо), который содержит общие поля (название, описание, калории, цена) и **абстрактные методы** (`displayInfo()`, `isAllergenFree()`).
    * Реализована иерархия классов-наследников: `Appetizer` (Закуска), `MainCourse` (Основное блюдо), `Dessert` (Десерт).
2.  **Полиморфизм:** Методы `displayInfo()` и `isAllergenFree()` **переопределены** в классах-наследниках для реализации специфической логики (например, проверка на глютен, лактозу, орехи).
3.  **Инкапсуляция:** Доступ к полям классов осуществляется строго через публичные методы `get*` и `set*`.
4.  **Класс-Группа (Агрегация):** Класс `RestaurantMenu` агрегирует и управляет списком объектов `Dish`.

#### Дополнительный Функционал

* **Сериализация/Десериализация:** Класс `FileWork` обеспечивает сохранение и загрузку всего объекта `RestaurantMenu` в файл, что позволяет сохранять состояние приложения между сеансами.

---

### 3. Функционал Приложения

Проект реализован как **консольное Java-приложение** с интерактивным меню (класс `Menu`).

| Пункт Меню | Описание Функционала | Демонстрируемый Принцип |
| :--- | :--- | :--- |
| **1. Добавить блюдо** | Создание и добавление нового объекта (`Appetizer`, `MainCourse`, или `Dessert`) в меню. | Наследование |
| **2. Посмотреть меню** | Выводит полную информацию о всех блюдах. | Агрегация |
| **3. Найти блюдо** | Поиск блюда по названию. | Инкапсуляция |
| **4. Удалить блюдо** | Удаление выбранного блюда из коллекции. | Класс-группа |
| **5. Сохранить меню** | Запись текущего состояния меню в файл. | Сериализация |
| **6. Загрузить меню** | Восстановление меню из файла. | Десериализация |
| **7. Блюда для аллергиков** | Фильтрация и вывод блюд, не содержащих аллергены. | **Полиморфизм** (`isAllergenFree`) |


### 3. Результаты

Проект полностью соответствует поставленной цели и решает проблему, демонстрируя понимание и практическое применение принципов **наследования, полиморфизма и инкапсуляции**. Реализован устойчивый к ошибкам консольный интерфейс и дополнительный функционал сохранения данных (сериализация).
